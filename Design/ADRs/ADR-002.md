# ADR-002: E2E Encryption Model

## Status
Superseded by ADR-004

## Context
ConnText needs an E2EE model that works for both 1:1 and group conversations.

## Decision
Use the Signal protocol for 1:1 conversations and the Megolm protocol for group conversations. The Signal Protocol is a well-established protocol that provides strong security guarantees, while the Megolm protocol is designed for efficient group messaging.

## Alternatives Considered
- **Use Signal protocol for both 1:1 and group conversations:** Rejected because the Signal Protocol is not optimized for large groups and can be inefficient in terms of bandwidth and storage.
- **Use Megolm for both 1:1 and group conversations:** Rejected because the Megolm protocol does not provide the same level of security guarantees as the Signal Protocol for 1:1 conversations.
- **Use Message Layer Security (MLS) protocol:** Rejected because the MLS protocol is still in development and not widely adopted, which could lead to compatibility issues and lack of support.
- **Use a custom E2EE protocol (libsodium):** Rejected because developing a custom protocol would require significant resources and expertise, and could introduce security vulnerabilities if not implemented correctly.

## Consequences
- The Signal protocol will be used for 1:1 conversations, providing strong security guarantees.
- The Megolm protocol will be used for group conversations, providing efficient messaging for large groups.
